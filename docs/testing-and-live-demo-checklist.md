## Emergency Fallback Checklist for the Event

**Deployment failure:**
- [ ] Have local dev server ready as backup
- [ ] Have hotspot ready if venue Wi-Fi fails
- [ ] Have printed case content as ultimate fallback
- [ ] Have facilitator notes with all passwords and solutions

**App crashes or bugs:**
- [ ] Have facilitator guide with all evidence text
- [ ] Be prepared to run game with printed materials only
- [ ] Have simple paper-based voting system for final accusation
- [ ] Have solution explanation ready to read aloud

**Network issues:**
- [ ] Test app with poor signal beforehand
- [ ] Verify case.json loads quickly over slow connection
- [ ] Have offline copy of case content available
- [ ] Consider using local network if internet fails

**Device compatibility issues:**
- [ ] Have spare devices available (old phones, tablets)
- [ ] Have printed materials for participants with incompatible devices
- [ ] Be prepared to share devices if needed
- [ ] Test on oldest expected device beforehand

**Participant confusion:**
- [ ] Have clear, simple instructions ready
- [ ] Have visual guide (screenshots) if possible
- [ ] Be prepared to walk participants through basic navigation
- [ ] Have FAQ ready for common questions

**Time running out:**
- [ ] Have accelerated version ready (skip some evidence)
- [ ] Be prepared to give hints to speed up progress
- [ ] Have facilitator reveal solution if time runs out
- [ ] Ensure debrief can happen even if game incomplete

**Data loss:**
- [ ] Remind participants that progress is local to device
- [ ] Encourage teams to discuss out loud (not just in app)
- [ ] Have facilitator track which team is on which chapter
- [ ] Be prepared to manually unlock chapters for teams if needed
